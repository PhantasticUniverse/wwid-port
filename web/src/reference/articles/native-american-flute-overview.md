# Native American Flute Overview

## Scope

This page is the encyclopedia front door for agents. It defines the working
scope of the local NAF encyclopedia, the major anatomy and workflow areas, and
the claim boundaries that route questions to deeper pages.

## Agent Summary

This repository is no longer only a Flutopedia mirror. It is an agent-first
NAF encyclopedia built from Flutopedia, governed local source intake, reference
books, WIDesigner documentation, CAD/tuning exemplars, research reports,
stress tests, and machine-readable artifacts. Use this overview to choose the
right detailed page before answering design, playing, manufacturing, safety,
or source-governance questions.

A Native American flute question can be about mechanism, music practice,
maker workflow, player fit, cultural context, legal/marketplace wording,
source rights, or measured release. Those are different claim types. The
correct answer may be a design path, a measurement requirement, a refusal, or
a private-only source boundary.

## Maker Workflow

1. Identify the question type: taxonomy, geometry, acoustics, fingering,
   playing, materials, manufacturing, validation, pedagogy, or governance.
2. Route to the most specific stable page before reading dated Q&A reports.
3. Check evidence class and source limits before making any public, safety,
   cultural, legal, measured, or manufacturing claim.
4. If the answer depends on source text, recordings, notation, scans, private
   CAD, measurements, or cultural context, use the source-governance page.
5. If the answer affects a physical flute, require datum, units, measured
   state, pressure, environment, fingering, and release decision.
6. If the answer affects people, groups, health, teaching, or accessibility,
   require declared user envelope and conservative claim language.

## Data Fields

Useful top-level fields include claim type, page id, source id, evidence class,
local path, rights status, CARE status, training status, mechanism class,
geometry state, fingering system, player envelope, material system, process
state, measurement state, public-release tier, and open-gaps status.

## Do Not Overclaim

Do not answer from a single page when a question crosses design, safety,
governance, and human-use boundaries.

Do not treat a mirrored page, private file, source-intake note, or generated
report as permission to publish text, notation, audio, images, geometry,
embeddings, or training data.

Do not say the encyclopedia is a validated predictive design engine. It is a
governed knowledge layer with many measured-data and fixture gaps still open.

## Open Gaps

Needed work includes maintaining strict zero-orphan report routing, reducing
planned and broken local-reference debt, aliases and redirects, source-catalog
page links, measured datasets, source-health validators, schema-aware artifact
validation, and page front matter for canonical titles, claim types, and
evidence classes.
